## БРИФ ДЕМО-САЙТА (можно отдать агенту как ТЗ)

### Общее
- Тип: одностраничный лендинг (one-page), плавный скролл к секциям.
- Бизнес: вымышленная кофейня «Зерно» (название/бренд можно заменить).
- Цель сайта: показать атмосферу + меню + привести к действию (заявка/бронь/как найти).
- Главное действие (CTA): «Забронировать столик» или «Связаться» (форма + контакты).

### Технологии и ограничения
- Чистые HTML + CSS + JavaScript (без тяжёлых фреймворков). Можно один маленький слайдер при необходимости.
- Семантическая верстка (header, nav, main, section, footer).
- Адаптив (mobile-first), корректно от 360px до десктопа.
- Быстрая загрузка, оптимизированные изображения, lazy-load для картинок.
- Доступность: контраст текста, alt у картинок, кликабельные области ≥ 44px.

### Дизайн-направление (можно заменить на свой стиль)
- Настроение: тёплое, уютное, «крафтовая кофейня».
- Палитра (пример): тёмно-коричневый/эспрессо, кремовый/беж, акцент — карамель/терракот.
- Шрифты: заголовки — характерный гротеск или мягкий serif; текст — читаемый sans-serif.
- Много «воздуха», крупные фото, аккуратная типографика.

### Секции (детально)
1. **Шапка (header/nav):** логотип «Зерно», меню-якоря (Меню, О нас, Галерея, Отзывы, Контакты), кнопка CTA. На мобиле — бургер-меню.
2. **Hero (первый экран):** крупный заголовок (напр. «Кофе, который будит город»), подзаголовок (1–2 строки), кнопка CTA, фоновое фото кофе/интерьера. Лёгкое затемнение фона для читаемости текста.
3. **О нас / Преимущества:** короткий абзац о кофейне + 3–4 карточки с иконкой («Своя обжарка», «Зёрна Specialty», «Уютная атмосфера», «Завтраки весь день»).
4. **Меню:** сетка карточек (фото + название + краткое описание + цена). 6–8 позиций (капучино, раф, фильтр, круассан и т.д.). Можно вкладки-категории (Кофе / Десерты).
5. **Галерея:** сетка из 6–8 фото (интерьер, напитки, выпечка). Аккуратная сетка, при клике — увеличение (lightbox) опционально.
6. **Отзывы:** 3 карточки с именем, оценкой (звёзды) и текстом. Можно простой слайдер.
7. **Контакты + форма + карта:** адрес, часы работы, телефон; форма (имя, телефон/email, сообщение) с валидацией; встроенная карта (заглушка/iframe). Кнопки соцсетей.
8. **Подвал:** лого, копирайт, соцсети, ссылка наверх.

### Поведение формы
- Валидация на стороне клиента (обязательные поля, формат email/телефона).
- Отправка: заглушка (mailto:) или комментарий «здесь подключается почта/Telegram-бот» — реальную интеграцию делаем под клиента.
- После отправки — сообщение «Спасибо, мы свяжемся с вами».

### Адаптив
- Mobile-first. Бургер-меню на телефоне. Сетки меню/галереи перестраиваются в 1 колонку.
- Проверить на 360px, 768px, 1280px.

### Перформанс (ВАЖНО — помни урок с FPS!)
- НЕ использовать тяжёлые «глянцевые/стеклянные» эффекты, обильные blur/backdrop-filter, тяжёлые тени на множестве элементов — это роняет FPS, особенно на телефоне.
- Анимации только лёгкие (fade/translate на скролле), через transform/opacity.
- Изображения сжаты (webp), заданы размеры, lazy-load.

### SEO (базовое)
- `<title>` и `<meta name="description">`.
- Один `<h1>`, логичная иерархия заголовков.
- Open Graph теги (og:title, og:image) — опционально.
- alt у всех картинок.

### Структура файлов (пример)
```
zerno-demo/
  index.html
  css/styles.css
  js/main.js
  img/ (оптимизированные изображения)
```

### Чего избегать
- Тяжёлых GPU-эффектов (см. перформанс).
- Лишних библиотек ради одной мелочи.
- «Простыни» текста — лендинг продаёт визуалом и краткостью.

## Чек-лист готовности демо
- [ ] Открывается, все секции на месте
- [ ] Адаптив на телефоне (бургер, сетки в 1 колонку)
- [ ] Форма валидируется и показывает «спасибо»
- [ ] Картинки оптимизированы, нет лагов прокрутки
- [ ] Есть title/description/alt
- [ ] Выглядит так, что не стыдно показать клиенту
